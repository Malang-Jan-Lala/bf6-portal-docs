# Project Structure

This repository is organized as a clean public documentation source.

## Folders

docs/
Human-readable documentation pages.

data/
Cleaned machine-readable reference data used for API/search/reference tooling.

examples/
Reserved for future original examples and reusable patterns.

tools/
Reserved for future documentation tools, API viewers, generators, and check scripts.

## Important idea
This repository is meant to become both:
* a human-readable handbook
* a source for a searchable API/reference viewer

The handbook explains concepts.
The viewer helps search exact functions, blocks, and enum values.

## Private content
Private/raw research should not be placed here unless reviewed.
Keep raw EA files, copied community scripts, screenshots, temporary files, and unfinished experiments outside the public documentation structure.
