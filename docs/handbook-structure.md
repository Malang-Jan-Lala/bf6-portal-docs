
# Handbook Structure

This page describes the planned structure for the future human-readable BF6 Portal handbook.

## Goal

The handbook should help creators understand concepts first, then use the API/reference viewer for details.

Simple idea:

```text
Handbook pages explain concepts.
API/reference viewer helps search exact functions, blocks, and enum values.
```

## Planned sections

Possible future handbook structure:

```text
Home
├── Getting Started
├── Rules Editor Model
├── API Reference
├── Selection Lists / Enums
├── Objectives
├── Player
├── AI
├── UI / Messages
├── Audio / Music
├── Examples
├── Glossary
└── Changelog
```

## Not the same as the API viewer

The API viewer is for searching details.

The handbook is for learning how things fit together.

Both should work together.

## Spatial / Godot note

Spatial/Godot map layout topics are planned, but not complete yet.

Examples:

* spawnpoints
* capture points / flags
* HQ / deploy objects
* vehicle spawns
* map layout
* Spatial JSON export/import

These should be documented separately when they are better understood.
